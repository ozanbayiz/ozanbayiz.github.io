<a id="setup" />
## Problem Setup

The environment is Push-T, a 2D control task where an agent pushes a T-shaped block into a goal zone.
Both policies use action chunking: at each decision step, the policy predicts $K = 8$ consecutive actions and executes them open-loop. The training dataset consists of expert demonstrations with z-score normalized states and actions.

<SectionDivider />

<a id="results" />
## Results

The core question: MSE behavioral cloning learns $\mathbb{E}[\mathbf{A} \mid \mathbf{o}]$, averaging over all modes of the action distribution. Flow matching samples from the full conditional $p(\mathbf{A} \mid \mathbf{o})$, producing trajectories that stay within a single mode. The difference is visible in the rollouts — the MSE agent wiggles indecisively while the flow agent maintains sustained contact and pushes the block into the goal region.

<a id="mse" />
## MSE Policy

The MSE policy predicts a single action chunk per observation by minimizing mean squared error against the expert:

$$
\mathcal{L}_{\text{MSE}} = \mathbb{E}_{(\mathbf{o}, \mathbf{A}) \sim \mathcal{D}} \left[ \| f_\theta(\mathbf{o}) - \mathbf{A} \|^2 \right]

$$

Implemented as a 3-layer MLP (hidden dims 256, 256, 256) with ReLU activations. The 5-dimensional state maps to $K \times 2 = 16$ outputs, reshaped to 8 two-dimensional actions. Trained with Adam ($\text{lr} = 3 \times 10^{-4}$) for 400 epochs.

<SectionDivider />

<a id="flow" />
## Flow Matching Policy

Flow matching defines a conditional probability path from noise to the expert action distribution. For each expert action chunk $\mathbf{A}_1$ and noise sample $\mathbf{A}_0 \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$, a neural network learns to predict the velocity $\mathbf{v}_\theta(\mathbf{o}, \mathbf{A}_\tau, \tau) \approx \mathbf{A}_1 - \mathbf{A}_0$ along the interpolation path $\mathbf{A}_\tau = \tau \mathbf{A}_1 + (1 - \tau) \mathbf{A}_0$.

At inference, action chunks are generated by solving the ODE from $\tau = 0$ to $\tau = 1$ with 10 Euler integration steps, starting from Gaussian noise. Same architecture and optimizer as the MSE policy, with the input extended to include the noisy action chunk and timestep (dim 22).
